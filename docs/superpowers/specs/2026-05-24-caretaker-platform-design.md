# 养护员小程序设计方案

**日期:** 2026-05-24
**项目:** FarmStar 养护员管理平台
**状态:** 已确认

---

## 1. 技术架构

### 1.1 项目形态
- **独立 uniapp 项目/小程序**，与 FarmStar 用户端完全分离
- 共享 `src/types/garden.ts` 中的类型定义和数据结构
- 复用现有 `gardenData.ts` mock 数据用于演示
- 复用现有组件样式（田园绿主题、卡片式布局）

### 1.2 技术约束
- 框架: uniapp + Vue 3 + TypeScript
- 样式: Tailwind CSS（与现有项目一致）
- 页面路由: pages.json（独立配置）
- 状态管理: Vue Composition API (ref/reactive)

### 1.3 登录系统
- **独立账号体系** — 养护员有单独的账号（手机号+验证码）
- 登录态独立管理（separate storage key: `caretaker_token`）
- 登录后获取 caretaker_id，后续 API 请求携带此 ID

## 3. 核心交互流程

### 3.1 登录流程
1. 养护员打开小程序 → 登录页
2. 输入手机号 → 获取验证码 → 验证通过后登录
3. 登录成功后跳转到首页（任务列表）

### 3.2 每日工作流（任务列表）
1. 打开 app → 进入首页任务列表
2. 看到今日待养护的田地清单（按优先级/紧急程度排序）
3. 点击田地 → 进入养护记录页
4. 完成养护动作 → 提交记录

### 3.3 扫码快速记录
1. 点击"扫码"入口 → 打开扫码界面
2. 扫描田地二维码 → 直接跳转到该田地的养护记录页
3. 选择动作类型 → 可选填写备注 → 提交

### 3.4 手动养护记录
1. 从任务列表或地图选择田地
2. 进入田地详情页 → 点击"养护记录"
3. 选择动作类型（浇水/除草/施肥/巡检/其他）
4. 填写备注（可选）
5. 提交记录

### 3.5 田地地图
1. 点击地图 tab → 查看所有田地分布
2. 点击地图标记 → 显示田地基本信息
3. 点击"进入养护" → 进入该田地养护页面

## 4. 页面详细说明

### 4.1 登录页 (caretaker-login)
- 手机号输入框（11位数字校验）
- 验证码输入框（6位数字）
- 获取验证码按钮（60秒倒计时）
- 登录按钮
- 错误提示（手机号格式错误、验证码错误等）

### 4.2 首页/任务列表 (caretaker-home)
- 顶部：养护员头像 + 姓名 + 今日任务数
- 筛选栏：全部 / 待养护 / 已完成
- 任务卡片列表：
  - 田地名称、作物信息
  - 认养用户数量
  - 最近养护时间
  - 紧急程度标识（如"三天未养护"标红）
- 底部导航：首页 / 地图 / 扫码 / 我的

### 4.3 田地地图 (caretaker-map)
- 全屏地图（Leaflet 或 uniapp map）
- 田地位置标记（不同颜色表示状态）
- 点击标记 → 弹出田地基本信息
- 底部快捷操作：进入养护 / 查看详情

### 4.4 田地详情页 (caretaker-field)
- 田地图片、名称、编号、面积
- 当前状态（空闲/已认养/待收获/养护中）
- 认养用户列表（头像+昵称）
- 监控预览（最新照片/视频）
- 养护记录列表（时间线形式）
- 快捷操作：开始养护 / 查看全部记录

### 4.5 养护记录页 (caretaker-care-log)
- 田地信息展示
- 动作类型选择（浇水/除草/施肥/巡检/其他）
- 备注输入框
- 提交按钮
- 历史记录列表

### 4.6 扫码页 (caretaker-scan)
- 全屏扫码界面
- 扫码成功后的跳转处理
- 扫码失败提示

### 4.7 我的页面 (caretaker-profile)
- 养护员头像、姓名、村庄
- 统计数据（负责田地数/累计养护次数/好评率）
- 设置入口
- 退出登录按钮

## 5. 数据流设计

### 5.1 API 层
- 复用现有 `gardenApi.ts` 服务层
- 新增养护员 API：
  - `caretakerLogin(phone, code)` — 养护员登录
  - `getCaretakerFields(caretakerId)` — 获取养护员负责的田地
  - `getCaretakerTasks(caretakerId)` — 获取今日养护任务
  - `createCareLog(fieldId, caretakerId, action, note)` — 创建养护记录
  - `getCareLogs(fieldId)` — 获取养护记录列表

### 5.2 数据过滤
- 所有田地查询通过 `caretakerId` 过滤
- 养护员只能操作自己负责的田地
- 认养用户信息对养护员只读

### 5.3 状态管理
- 登录态：`caretaker_token` 存本地
- 当前养护员信息：`caretakerInfo` (reactive)
- 田地列表：`fields` (ref)
- 养护任务：`tasks` (ref)

## 8. 验收标准

1. 养护员可以通过手机号+验证码登录
2. 登录后看到自己负责的田地列表
3. 可以查看田地详情、认养用户、监控画面
4. 可以通过扫码快速选择田地
5. 可以记录养护动作（浇水/除草/施肥/巡检等）
6. 可以查看历史养护记录
7. 可以切换到地图视图查看田地分布
8. 视觉风格与用户端保持一致
